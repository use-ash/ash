# Session Start Checklist

- [ ] Load the persistent conversation state file.
- [ ] Read `session_metadata` to see what happened most recently.
- [ ] Review `global_context` for shared priorities or blockers.
- [ ] Review every active thread title, summary, and next step.
- [ ] Match the current user request to the correct thread.
- [ ] Load any relevant memory notes for that thread.
- [ ] Confirm that the selected thread still has a valid next step.
- [ ] Repair the thread summary if the stored summary is stale or incomplete.
- [ ] Start the task only after the working context is clear.
