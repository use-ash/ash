Use this file as a starter template for your own `MEMORY.md` index.

Replace the example topics with your real notes.

# Memory Index

This file is the short index for durable memory.

Load this file at the start of each session.
Load individual notes on demand.
Do not turn this file into a transcript.

## How To Use This Index

- Read this file first.
- Load referenced notes that match the current task.
- Verify memory against current source files or state before acting.
- Update canonical notes instead of creating duplicates.

## Index design rules

Keep the real index:

- short enough to load every session
- organized by importance, not by creation date
- limited to canonical notes
- free of duplicate links

When the note count grows, the index should still route the agent. It should not try to summarize every note in the system.
